A common frustration I come across when working with Angular (or any other framework for that matter), is running functions based on key bindings. The typical method of doing this is unreadable and messy. For example, if you wanted to run a function when a user hit CTRL+ALT+D your code would look something like this.

True, it could be worse. Never the less, doing this can become very messy very quickly. It is also difficult to quickly find and fix problems because you have to know exactly which key code is which to know what key command is being listened for. In a recent project I was working on, I decided to remedy the issue by abstracting the proccess into an Angular service and now I'm going to show you how to do it.

In the same folder as your service, create two new files: one called modifiers.ts and one called keycodes.ts and copy the following code into them. You can move these later, but for now it is easier to keep them in the same folder as your service.

These files contain helpers we will use to make our service work properly. For example keycodes.ts has a list of all the key codes, mapped to readable names, and modifiers.ts has a helper function to check if one or more modifier keys are present in and event.

Theoretically, they already make your life easier. You could use them as below.

This code is much cleaner, but we can do better than that. What we want is a declarative way of listening to key events.

In the key bind service you will create a new method called match(). We will have parameters so that you can specify the keys and modifiers you want to listen for, an optional EventTarget so that you can specify where to listen for the events, and return an Observable that fires when the binding is matched.
